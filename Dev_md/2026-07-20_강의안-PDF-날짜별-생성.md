# 2026-07-20 · 수강생 배포용 강의안 PDF 날짜별 생성

## 한 일

`scripts/generate-handouts.mjs` 신규 작성 — `src/data/courses.ts`를 단일 원본으로 삼아
수강생 배포용 인쇄물 4종을 `handouts/`에 생성한다.

| 파일 | 내용 | 분량 |
| --- | --- | --- |
| `00_동신대학교_AI연수_표지` | 표지 + 연수 개요 + 강의안 사용법 + 전체 일정(대면 3일·VOD 4모듈) + 3일 학습 여정 | 3쪽 |
| `01_Day1_강의안` | Day 1 속표지 + 오늘의 흐름 + 1~4교시 상세 | 8쪽 |
| `02_Day2_강의안` | Day 2 동일 구조 | 8쪽 |
| `03_Day3_강의안` | Day 3 동일 구조 | 8쪽 |

각 포맷을 함께 굽는다 — `.pdf`(배포용) / `.html`(원본) / `.doc`(Word 편집본).

교시 블록 구성: 교시·시간·난이도·중요도 배지 → 학습 목표 → 주요 내용(`**굵게**`·`==형광펜==`
인라인 마크업 반영) → 실습 프롬프트 전문(복사 가능한 완성형).

## 왜 이렇게 했나

- **날짜별 3권 분리**: 대표 요청. 수강생이 수업 당일 해당 권만 지참하면 되게 함.
- **courses.ts 단일 원본**: 강의안을 따로 손으로 쓰면 사이트 콘텐츠와 어긋난다.
  사이트를 고치고 스크립트를 다시 돌리면 인쇄물이 자동으로 따라온다.
- **표지 로고는 `logo-horizontal-ko.png`**: `dsu-signature-ko.png`는 1417×2127 세로 시그니처
  **시트**라 표지 상단에 넣으면 로고 여러 개가 겹쳐 보인다(1차 출력에서 실제로 그렇게 나옴).
- **`.block`에 `break-inside: avoid-page`를 걸지 않음**: 교시 블록이 한 쪽을 넘는 분량이라
  통째로 묶으면 앞 페이지 절반이 빈 채로 넘어간다. 대신 머리말(`break-after: avoid`)과
  목표·항목 단위만 쪼개지지 않게 고정.

## 사고 / 막힌 지점

- **Word 변환에서 표 전부 유실**: macOS `textutil -convert docx`는 HTML의 `<table>`을
  하나도 옮기지 않는다(최소 예제로 재현 확인 — `<w:tbl>` 0개). 텍스트만 남아 일정표·개요표가
  통째로 사라졌다. → `.docx` 포기하고 **Word가 직접 읽는 HTML 기반 `.doc`**으로 전환.
  Word는 표·색상·형광펜까지 그대로 열어 편집할 수 있다.
  CSS 변수는 Word가 해석 못 하므로 `var(--accent)`를 실제 색상값으로 치환해 굽는다.
- **puppeteer는 package.json에 넣지 않음**: OG 이미지 스크립트의 sharp와 같은 방식(CLAUDE.md §4).
  CI를 가볍게 유지하려고 임시 설치로만 쓴다.

## 사이트 연결 (좌측 과정 메뉴 상단 다운로드)

`CourseCategory.tsx` 사이드바 맨 위에 **강의안 다운로드** 블록 추가 —
그 날짜의 `PDF` · `워드` + `표지 · 전체 일정` 3개 링크.

- 배포본 경로: `public/handouts/dsu-<cover|day1|day2|day3>.pdf|.doc`
  (스크립트가 `handouts/`에서 자동 복사. **한글 파일명은 gh-pages 경로 인코딩 사고를 부르므로
  웹 배포본만 ASCII 슬러그**로 두고, `download` 속성으로 받을 때 한글 이름이 되게 함)
- 공통 VOD 페이지는 강의안이 없으므로 블록 자체를 숨김(`hasHandout`).
- 스타일 `.course-sidebar-dl-btn` (`src/styles/programs.css`) — 색상은 전부 CSS 변수라
  다크모드·과정별 컬러가 자동으로 따라온다.

## 실행법

```bash
npm i -D puppeteer            # 최초 1회 (CI 경량 유지를 위해 package.json 미등재)
npm run handouts              # = node scripts/generate-handouts.mjs
```

다른 위치에 설치된 puppeteer/Chrome을 쓰려면:

```bash
PUPPETEER_DIR=/경로/node_modules/puppeteer/lib/puppeteer/puppeteer.js \
PUPPETEER_EXECUTABLE_PATH="$HOME/.cache/puppeteer/chrome/<버전>/chrome-mac-x64/Google Chrome for Testing.app/Contents/MacOS/Google Chrome for Testing" \
node scripts/generate-handouts.mjs
```
