## 1. 다크모드 가독성

### 무엇이 문제였나 (실측)

헤드리스 크롬으로 전 텍스트 노드의 대비를 재서 확인했다(반투명 배경은 부모 위에 합성해 계산 —
합성하지 않으면 멀쩡한 곳까지 미달로 잡히는 오탐이 대량 발생한다).

- `practice.css`에 **밝은 배경이 하드코딩**되어 있었고 다크 오버라이드가 없었다.
  흰 배경(`#fff`)이 다크에서도 그대로 남아 밝은 글자가 얹혀 **대비 1.12:1** — 사실상 안 보이는 상태.
  `#E8F5E9`(연녹) 카드 위 밝은 글자는 **1:1**.
- `CourseCategory`가 과정 색을 **인라인 CSS 변수로 주입**하는데, 인라인은 `dark-mode.css`보다
  우선한다. 그래서 다크에서도 라이트용 진한 파랑(`#1565C0`)이 쓰여 어두운 배경 위 어두운 글씨(2.93:1).
- 다크에서 `--primary-blue`는 밝은 파랑(`#8FACE6`)인데 채워진 버튼의 글자는 흰색이라 **2.28:1**.

### 어떻게 고쳤나

- `base.css`에 **시맨틱 색 변수** 도입(`--success-*` / `--warn-*` / `--danger-*`),
  `dark-mode.css`에 다크 대응값(배경은 반투명으로 낮추고 글자는 밝게). `practice.css`의
  하드코딩 색 34종을 전부 변수로 치환.
- `CourseCategory`에 **테마별 과정 색**(`DARK_TINT`)을 두고, 인라인으로 원색을 쓰던 곳
  (배지·패들릿 버튼·번호 원·아이콘)도 전부 테마 색을 쓰게 통일.
- 다크에서 밝은 색으로 채워진 버튼은 글자를 어두운 배경색(`#0E1422`)으로 뒤집음.

### 결과

`/courses/day1` · `/courses/day3` · `/prompt-eval/workshop` · `/prompt-eval/practice`
**다크 4개 페이지 대비 미달 0건**(WCAG AA 기준). 라이트도 회귀 없음.

### 사고 — CSS 주석을 깨뜨려 규칙이 통째로 무시됨

주석 블록을 편집하다 `*/`가 중복돼 뒤따르는 규칙 전체가 파싱에서 날아갔다.
빌드는 성공하고 화면만 안 고쳐져서 원인 찾는 데 시간이 걸렸다.
**CSS는 문법 오류가 조용히 넘어간다** — 고쳤는데 반영이 안 되면 주석/중괄호부터 볼 것.

### 사고 — 일괄 치환이 텍스트 색까지 배경 변수로 바꿈

`#fff` → `var(--card-bg)` 일괄 치환에서 **`color: #fff`(색 있는 버튼 위 흰 글자)** 까지 바뀌었다.
`color:` 속성만 골라 `var(--text-white)`로 되돌림. 색상 일괄 치환은 배경/글자 용도를 나눠서 할 것.

## 2. 영문 전환 (KO/EN)

### 무엇이 문제였나

- 언어 전환이 **우측 톱니(설정) 드롭다운 안에만** 있어 찾기 어려웠다.
- `useState('ko')`뿐이라 **새로고침하면 한국어로 되돌아갔다**. "자동변환도 안 된다"는 제보의 실체.
- 영문 데이터가 **과정명·개요·학습자료까지만** 있고, 정작 핵심인
  **커리큘럼 세션 상세(학습목표·주요내용·실습 프롬프트 72개)와 실습 랩에는 영문이 전혀 없었다**.

### 어떻게 고쳤나

- **KO/EN 스위치를 상단 메뉴에 상시 노출**(`.nav-lang-switch`), 선택은 `localStorage('dsu-language')`에
  저장. 저장값이 없으면 브라우저 언어로 시작(한국어가 아니면 영어). `<html lang>`도 함께 갱신 —
  스크린리더와 브라우저 번역기가 이 값을 본다.
- **영문 오버레이 도입** — `src/data/i18n/`에 `vod-en.ts` · `day1-en.ts` · `day2-en.ts` ·
  `day3-en.ts` · `handson-en.ts`. 키는 `<programId>:<curriculumDay>:<sessionIndex>`.
  번역 규모는 세션 29개 · 실습 프롬프트 71개 · 랩 12개(39단계).
  - **왜 인라인 `~En` 필드가 아니라 오버레이인가**: `courses.ts`는 이미 98KB다. 세션마다 영문을
    끼우면 한국어 원본을 읽고 고치기 어려워진다. 강의 콘텐츠는 한국어 기준으로 수시로 손보므로
    원본은 원본대로 두는 편이 유지보수에 낫다.
  - **키가 없으면 한국어 원문으로 자동 폴백** — 번역 누락이 빈 화면이 되지 않게.
- 프롬프트 라벨은 구조를 유지한 채 번역: `[역할]`→`[Role]`, `[맥락]`→`[Context]`,
  `[요청]`→`[Request]`, `[형식]`→`[Format]`, `[예시]`→`[Example]`, `[조건]`→`[Condition]`.
  `[대괄호]` 빈칸도 영문 플레이스홀더로(`[과목명]`→`[course name]`).
- 정형 라벨은 규칙 변환(`labels.ts`): `모듈 1`→`Module 1`, `1~2교시`→`Periods 1–2`,
  `60분`→`60 min`, 난이도 `기초`→`Foundational`.
- `PromptBlock`의 "프롬프트 / 복사" 라벨도 언어 대응.

### 판단이 필요했던 지점

바이브코딩 실습 프롬프트에 **"전부 한국어로"** 라는 지시가 들어 있었다. 이건 교수가 만드는
**수업 도구의 출력 언어** 지시라 그대로 번역하면 영어권 교수가 한국어 도구를 만들게 된다.
→ `in the language your students use (Korean or English)`로 바꿨다.
동신대 학생 대상 도구라면 한국어가 맞을 수 있어 선택 여지를 남기는 표현으로 둔 것이다.
